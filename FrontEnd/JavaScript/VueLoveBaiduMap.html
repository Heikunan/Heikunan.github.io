# 百度地图密集点渲染开发的三种思路

# 黑铁时代-百度地图原生开发

所谓的原生地图开发，指的是在一个 html 文件中的 <script></script> 标签中，不断的堆叠地图的代码，如果项目比较复杂的话，就像三年前 jQuery 时代，写一大堆意大利面条式的代码，非常难以维护，以现代前端开发的角度来看，也不会采用这种思路，而且如果要渲染大量的自定义的点的话，要使用百度地图的 new Bmap.Marker() 方法，数量过大的话，浏览器会出现卡顿的情况

# 青铜时代-函数驱动地图渲染-工厂模式

其核心的方法是在百度地图核心的 Vue 文件中写一个 initMap() 函数

methods:{
    initMap(){
      const map = new BMap.Map("mapcontainer", {
        enableMapClick: true
      });
      //doSomeThings~~~
    }
}

在每次数据改变的时候，调用 initMap 方法，实现百度地图的重绘

watch:{
  'getCurrectColorData'(){
      this.initMap()
  },
  'getMapPoints'(){
      this.initMap()
  },
  'getFencesPoints'(){
      this.initMap()
  },
  'getResetMap'(){
      this.initMap()
  }
}

initMap() 函数就像一个工厂，不断的重新去生产一个新的对象，如果在一个页面中，经常调用这个函数的化，会产生大量垃圾，造成资源没有必要的浪费，同时如果只是更新了部分的数据的话，其他数据没有发生改变，也会调用 initMap() ，这样会造成 CPU 做一些无用的计算，性能就会大大的降低。

# 白银时代-数据驱动地图渲染-单例模式

要做成数据驱动地图渲染必须满足一下几个条件

  1. 全局只创建一个百度地图对象
  2. 百度地图对象支持动态增加和删除子对象
  3. 数据改变时地图和视图绑定自动更新

首先在 BaiDuMap.vue 文件中提供一个地图容器

<template>
    <div id="mapcontainer"></div>
</template>

然后再借助 Vue 的生命周期中的 mounted() 钩子在 DOM 加载结束的时候挂上百度地图，同时将其挂载到 Vue.prototype 将其提升为一个全局的变量

mounted(){
    const map = new BMap.Map("mapcontainer", {
        enableMapClick: false
    });
    Vue.prototype.map = map
    //doSomeThings
}

这样就可以在各个函数中操作百度地图了

methods:{
    mapAddOverlay(mapPoints = []){
        for(let i in this.map.De){
            let infoBox = this.map.De[`${i}`]
            if(infoBox.hasOwnProperty(`_isOpen`) && infoBox.hasOwnProperty(`_content`)){
                infoBox.close()
            }
        }
        const geoc = new BMap.Geocoder();
        const allPoints = [mapPoints[0], mapPoints[mapPoints.length-1]]
        allPoints.map(v=>{
            geoc.getLocation(v,(rs)=>{
                const addComp = rs.addressComponents;
                const infowindowHtml = `
                    <div>
                        <div>
                            <div class='infowindow-item'>
                                <span>车牌号：</span><span>${v.CarData.carCode}</span>
                            </div>
                            <div class='infowindow-item'>
                                <span>信号点状态：</span><span>${v.CarData.gpslock}</span>
                            </div>
                            <div class='infowindow-item'>
                                <span>速度：</span><span>${v.CarData.speed}</span>
                            </div>
                            <div class='infowindow-item'>
                                <span>GPS时间：</span><span>${v.CarData.gpsTime}</span>
                            </div>
                            <div class='infowindow-bottom'></div>
                        </div>
                    </div>`
                const infowindow = new BMapLib.InfoBox(this.map,infowindowHtml,{
                    closeIconUrl:'/static/close.png'
                })
                const InfoWindowMarker = new BMap.Marker(rs.point);
                infowindow.open(InfoWindowMarker);
            })
        })
        this.map.removeOverlay(Exbigline)
        const overPoint = mapPoints
        const Exbigline = new BMap.Polyline(mapPoints, {
            strokeColor:"blue",
            strokeWeight:4,
            strokeOpacity:0.5
        });
        //增加折线
        this.map.addOverlay(Exbigline);
    }
}

把需要和数据绑定的覆盖物单独写到 BaiDuMap.vue 地图核心的 methods 中然后再用watch函数对其监听,此时的 initMap() 方法就变成处理轨迹点的方法了，把核心的逻辑分割成几个小的部分，方便后期的维护和修改

watch:{
  'getCurrectColorData'(){
      this.initMap()
      this.mapAddFencesPoints()
  },
  'getBaiduMapPoints'(){
      this.initMap()
  },
  'getFencesPoints'(){
      this.mapAddFencesPoints()
  },
  'getResetMap'(){
      this.initMap()
      this.mapAddFencesPoints()
  },
  'getStartAndEndPoints'(nextVal){
      this.mapAddOverlay(nextVal)
  }
}
